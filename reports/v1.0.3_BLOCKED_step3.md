## Diagnostics

Initial Step 1 diagnostic showed:

```text
[mixtral] metal_ctx: true, weights_mmap_buf: false, embed_buf: true, decode_arena: true
```

Root cause: Metal refused a single 25 GiB no-copy buffer over the full GGUF
mmap. The loader now pins the GGUF as overlapped no-copy chunks and stores
chunk-local offsets on each `TensorRef`.

After the fix:

```text
[mixtral] metal_ctx: true, weights_mmap_buf: true, embed_buf: true, decode_arena: true
```

## Likely Next Work

- Correctness first: output garbage likely points at a tensor orientation,
  RoPE layout, gate routing, or expert down projection issue.
- Speed second: the first working path is dominated by many per-layer GEMVs,
  CPU readbacks, CPU attention/routing, and final LM-head readback. It needs
  profiling before chasing the `>=3 dec_tps` target.
- Keep the chunked no-copy mmap approach; it solved the 25 GiB Metal buffer
  refusal without copying the whole GGUF into resident memory.
