## Performance note

Serial verify gives 1× speedup regardless of acceptance rate α — each accepted
draft still requires one forward pass for KV update. The 1.2-1.4× claim in spec
requires batched verify: K-token DecodeArena with per-token x_buf/q_buf
allocation and buffer-offset dispatch.

## Phase 5 recommendations

Based on TCB trace (tcb_commit=69.4%, gemv_f16=30.3%):

1. **LM head top-K filter** — compute logits only for top-K vocabulary tokens
   (e.g. K=8192). Targets the 30.3% gemv_f16 slice.
2. **Multi-layer fused TCB** — amortize the 69.4% tcb_commit overhead by
   fusing 2-4 layers into one command buffer. Already partially done (single-TCB
   per token); next step is reducing commit count across layers.
3. **True batched forward (K-token DecodeArena)** — K separate x_buf/q_buf GPU
   allocations + buffer-offset dispatch per layer. Required for 1.2-1.4× n-gram
   spec speedup. Unblocked by Phase 4D's n-gram infrastructure.
4. **Re-measure** after Phases 5A+5B land — bottleneck distribution will shift.
