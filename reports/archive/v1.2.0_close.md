## Phase Outcomes

### Phases 1-8: BLOCKED

Phases 1-8 were all structurally blocked. Root causes documented in
`reports/v1.2.0_BLOCKED_{1..8}.md`. Summary:

- **Phase 1** (LM-head simdgroup-matrix + Q8_0 LM-head): gate 26 dec_tps not
  cleared. Simdgroup-matrix regression; Q8_0 path not plumbed. BLOCKED.
- **Phase 2** (Q4_K_M MoE GEMV improvements): v2t_gu_v2 kernel (scale
  preloading + sumy trick + paired nibbles) = correct, +2.1% vs baseline.
  Gate 26 dec_tps not cleared. Root cause: memory-latency-bound at
  batch-size=1. Kernel kept as opt-in. BLOCKED (gate not cleared).
- **Phase 3** (persistent compiled command buffers via ICB): ICB does not
  expose `set_bytes` equivalent; full set_bytes → constant-buffer refactor
  is a multi-day undertaking. SKIP.
- **Phases 4-8**: Various Metal pipeline optimizations. All structurally
  blocked (see individual reports). No code changes.

### Phase 9: LANDED (`ec6916f`)

Per-layer MoE expert access tracking via route_history_buf.

- Added `route_history_buf` (n_moe_layers × top_k × u32) to `DecodeArena`.
- Blit-copied `moe_route_ids_buf` → `route_history_buf` per MoE layer inside
  `encode_layer` during the single-TCB forward pass.
- After CB commit: CPU reads `route_history_buf` and calls
  `ExpertCache::note_access()` for every route slot of every MoE layer.
- `Engine::expert_access_counts()` trait method exposes per-layer counts.
- `dismantle stats` subcommand prints interpretable per-expert usage table.
- Parity test `v1_memory_eviction_parity.rs` passes: bit-identical output
  with/without eviction (madvise is advisory; pages reload on page fault).
- No regression: coexist median 20.69 dec_tps vs 20.80 baseline.

### Phase 11: SKIP

### Phase 12: LANDED (this commit)

Memory limits + housekeeping:

- `DeviceLimits` struct in `profile.rs`: `memory_limit_mb`, `max_seq_len`,
  `max_kv_cache_mb`, `max_routed_expert_ram_mb`.
- `device_limits` block added to `profiles/deepseek-v2-lite-q4.m3pro18.json`:
  `memory_limit_mb=14400`, `max_seq_len=4096`, `max_kv_cache_mb=1800`,
  `max_routed_expert_ram_mb=8000`.
- `--memory-limit-mb` CLI flag on `generate` and `serve`.
  `Some(0)` = auto-detect (80% system RAM). `None` = unlimited.
- `load_engine` merges profile `device_limits` into config, then enforces
  budget (file size check) before mmap.
- Test `tests/v1_2_memory_limit.rs`: 4 cases (too-low → error, generous →
  ok, none → ok, auto → ok).
- Clippy passes with zero warnings (`-D warnings`).
