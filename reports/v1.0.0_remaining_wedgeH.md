**Path 2 (simdgroup_matrix): PASS in 2 iterations.**

Iteration 1: Activation tile layout wrong — used column index `k = elem & 7` as
x index, producing B[m][k] = x[c+k] (non-broadcast). Result was sum of weights
times x[j] instead of dot product.

Iteration 2: Fixed to row-broadcast — `shmem_x[elem] = x[c_base + m]` where
`m = elem >> 3` (row index). Matches quant.metal pattern. All 3 tests pass.

**New kernel: `gemv_simdgroup_f32` (`shaders/matmul.metal`)**
- One SIMD group (32 threads) per threadgroup; computes 8 output rows per group
- Grid: ceil(rows/8)×32; TG: 32
- Weight tile shmem_w[8][8]: W[base_row..+8][c_base..+8] f32
- Activation tile shmem_x[8][8]: broadcast — row m has x[c_base+m] in all columns
- (A×B)[i][j] = Σ_m W[row_base+i][c_base+m] × x[c_base+m] = partial GEMV ✓
- All columns of acc identical after broadcast; column 0 extracted for y output
- Requires cols % 8 == 0 (enforced in dispatcher)

**Wiring: Phase 2 mini-TCB (q_b_proj GEMV)**
- When `q_lora % 8 == 0 && q_out_rows % 8 == 0`: uses `gemv_simdgroup_f32_tcb`
- Fallback: `gemv_f32_attn_pinned_buf_tcb` (existing, uncounted)
- For DeepSeek-V2: q_lora_rank=1536, rows=n_heads×head_dim_q — both multiples of 8
