## Structural Notes

**Fusion scope: attention q_a + kv_a only.** The spec listed FFN fusion as a goal
(`rmsnorm_gemv_q4k_pair_pinned`), but FFN fusion requires computing rmsnorm twice ŌĆö
once for the gate logit GEMV (`x_norm_buf ŌåÆ moe_logits_buf`) and once for the MoE
block input. Net cost exceeds savings. FFN fusion is skipped; only attention fusion
(q_a + kv_a) lands in this wedge, eliminating 2 dispatches per layer.

**What changed in `attention_tcb_inner` Phase 1 mini-TCB:**
- Before: `gemv_f32_attn_pair_arena_tcb` (reads `arena.x_norm_buf` ŌĆö pre-normed)
- After: two `rmsnorm_gemv_f32_attn_pinned_tcb` calls (read `arena.x_buf` directly)
- The fused kernel runs variance reduction + GEMV in one threadgroup pass.
- `attn_norm_buf` now fetched inside `attention_tcb_inner` (guaranteed by `tcb_base`).

**What changed in Wedge C mini-TCB ╬▒:**
- Before: `[add_inplace(li>0)] + rmsnorm_attn(x_buf ŌåÆ x_norm_buf)` ŌĆö always a TCB commit
- After: `[add_inplace(li>0)]` only ŌĆö li==0 skips TCB entirely; li>0 emits one dispatch
- Saves ~27 standalone rmsnorm dispatches/token (one per layer, attention path).

**New dispatcher:**
- `rmsnorm_gemv_f32_attn_pinned_tcb`: TCB variant of the existing counted
  `rmsnorm_gemv_f32_attn_pinned_metal`. Bindings match the Metal kernel exactly
  (buf0=w, buf1=x, buf2=weight, buf3=eps, buf4=out, buf5=rows, buf6=cols).

**16-token greedy parity (model required):** Not run ŌĆö no model weights available.
Unit-level parity for all 3 Wedge G tests confirmed by `v1g_rmsnorm_gemv_fusion_parity`.
The `wedge_g_fused_pair_matches_cpu` test validates the exact two-call pattern used in
`attention_tcb_inner` (same x_buf, different w, same weight) at atol 1e-3.
