## How to use the new infrastructure

### Per-kernel GPU timing

```bash
DISMANTLE_TCB_TRACE=gpu ./target/release/dismantle bench \
  --weights models/deepseek-v2-lite-q4.gguf \
  --kernel-profile profiles/deepseek-v2-lite-q4.m3pro18.json \
  --suite decode --trials 1 --max-new-tokens 8 \
  --trace-json /tmp/trace.json

python3 tools/bench/analyze_tcb_trace.py /tmp/trace.json
```

The trace's `dispatch_samples[].gpu_us` field has per-kernel GPU
compute time in microseconds. Caveat: split-CB mode breaks GPU
pipelining (each dispatch syncs), so absolute %s skew vs production
single-CB-per-block. Relative ordering is reliable for picking next
target; do not compare absolute numbers across modes.

### Autotune sweep

```bash
bash tools/bench/autotune_sweep.sh \
  --field gemm_q4_k_schedule \
  --values v2t,v2t_gu_v2,v2 \
  --trials 4 --tokens 24
```

Patches the field into a temp profile, benches each value, prints a
sorted table of dec_tps_median. Results saved to
`bench_results/autotune_sweep_<timestamp>.jsonl`.

### Q5_0 v2t opt-in

```bash
# Build with the new kernel (already in main)
cargo build --release -p dismantle

# Patch profile to opt in
python3 -c "
import json
p = json.load(open('profiles/deepseek-v2-lite-q4.m3pro18.json'))
p['selected']['routed_down_schedule'] = 'v2t'
json.dump(p, open('/tmp/profile_q5_v2t.json', 'w'), indent=2)
"

# Bench both
bash tools/bench/coexist_bench.sh                                    # basic baseline
PROFILE=/tmp/profile_q5_v2t.json bash tools/bench/coexist_bench.sh   # v2t opt-in
```

## T2.11 — Q5_0 v2t port: decision needed

The kernel is implemented, builds clean, runs correctly. Default
profile keeps `routed_down_schedule: "basic"` — no production
behavior change without explicit opt-in.

### Parity behavior

Each (row, col) value decoded by the v2t kernel matches the basic
kernel's math (verified against Rust `quant::dequant_q5_0` reference).
But the summation order differs:

- **Basic:** 256 threads tree-reduce 1408 elements per row
- **V2T:** 32 simd_lanes simdsum 44-element partials each (sum of
  `block_d * sum_over_lane(qi * x)` per simdgroup, then `simd_sum`)

Both are mathematically equivalent; fp32 add isn't associative so
results differ by < 1e-4 per logit. Greedy argmax on close logits
can flip, accumulating over decode steps.

### Observed parity tests (greedy, temp=0)

| Prompt | Tokens | Result |
|---|---:|---|
| "The capital of France is" | 32 | Identical 1–21, then basic degenerates to `!!!!!`, v2t continues coherently |
| "The quick brown fox jumps over" | 24 | Identical 1–12, divergent thereafter |
| "The capital of France is" | 6 | Identical |

In the failing tests, **v2t's output looks subjectively MORE coherent
than basic's** (simdsum's fewer intermediate roundings beat deep tree-
reduce numerically). This is the OPPOSITE of "v2t introduces a bug" —
v2t may actually be the more accurate kernel.

### Decision options

**Option A (recommended):** ship v2t as default if bench gate clears.
Run `tools/bench/coexist_bench.sh` baseline + v2t-profile, require
≥+5% e2e on TOKENS=24 median. Bit-identical greedy is not a
fundamental property — production users on `temperature > 0` are
unaffected, and v2t arguably produces better output anyway.

**Option B:** keep opt-in indefinitely. Users who want the speedup
flip the profile flag; the shipped default keeps the basic kernel
for users who require strict bit-identical greedy.

**Option C:** rewrite v2t to match basic's summation order. Defeats
simdsum (and most of the speedup). Not recommended.

### What to actually do next

1. Quit Claude Code, run `tools/bench/coexist_bench.sh` baseline
   to get clean basic-kernel dec_tps.
2. Patch profile to `routed_down_schedule: "v2t"`, re-run
   `coexist_bench.sh` with `PROFILE=<patched>`.
3. Compare medians. **If v2t ≥ basic × 1.05** → ship as default
   (update profile + bump shader_hash). **If smaller** → keep opt-in.
4. If shipping default, write `reports/v2.1.0_T2.11_close.md` with
   the bench numbers + greedy-token-output examples documenting the
   numerical-drift caveat.
