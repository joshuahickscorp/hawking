## Counting methodology note

`stats.commits` is incremented ONLY by `MetalContext::dispatch_threads()` and
`MetalContext::dispatch_batch()` (the old per-kernel single-CB path). The
`TokenCommandBuffer::commit_and_wait()` does NOT increment this counter — TCB
commits are "uncounted" dispatches. The prompt's ~380 figure refers to all
`commit_and_wait()` calls per token, counted independently. With 27 layers and
roughly 7 TCBs/layer, the per-layer breakdown yields ~191 TCB commits for the
Wedge C path and ~218 for Wedge F.

## Consolidation plan (Wedge C, in order of impact)

### Consolidation 1: Fold post-loop (1831 + 1843 → 1 TCB)
**Saves:** 1 commit/token  
**Risk:** low  
**How:** merge the add_inplace and rmsnorm into a single TCB in the post-loop block.

### Consolidation 2: Fold Phase 3 (2190) + Mini-TCB β (1805) into one TCB per layer
**Saves:** 27 commits/token  
**Risk:** medium  
**How:** extract Phase 3 kernel encoding out of `attention_tcb_inner` (the function commits after Phase 2 and returns key params), then in the caller create one TCB that encodes Phase 3 + Mini-TCB β kernels.

### Consolidation 3: Fold gate GEMV (2260) into same TCB as Phase 3 + Mini-TCB β
**Saves:** 27 additional commits/token (total 54 for consolidations 2+3)  
**Risk:** medium (crosses ffn_tcb_inner boundary)  
**How:** after encoding Mini-TCB β into the combined TCB, pass the TCB into ffn_tcb_inner for gate encoding, then commit in ffn_tcb_inner before the CPU read.

### Consolidation 4: Fold embed (1774, li=0) and add_inplace (1790, li>0) into Phase 1 TCB
**Saves:** 27 commits/token  
**Risk:** medium (modifies attention_tcb_inner signature)  
**How:** add optional pre-phase encode into Phase 1's TCB construction in `attention_tcb_inner`.

### Wedge F consolidations (lower priority — non-default path)
- Fold 1644+1659+1679+1705 into per-layer combined TCB ending at gate (saves 3×27=81/token for Wedge F, but Wedge F is off by default)
- Skip for now to limit scope.
