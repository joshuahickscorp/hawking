Design:
- encode_gemv_f32_attn_pinned encodes one fp32 GEMV into a CommandBatch, mirroring
  encode_gemv_q4_k_m_simd but for fp32 weights (no quantization decode needed).
- dispatch_gemv_f32_attn_pinned_pair_batched coalesces TWO fp32 GEMVs (A and B) that
  share the same input x into ONE ctx.dispatch_batch call: one x_buf upload, two
  sequential GEMV compute passes in one CB, one GPU→CPU readback per output.
- gemv_f32_attn_pair_dispatch routes to the batched path when both pinned buffers are
  Some; falls back to two sequential gemv_f32_attn_dispatch calls otherwise.
- attention() hoists kv_a_dim/kv_a allocation before the q-lora / non-q-lora branch
  so both branches can coalesce their first GEMV (q_a_proj or q_proj respectively)
  with kv_a_proj_with_mqa into one CB. Standalone kv_a_proj_with_mqa dispatch deleted.
