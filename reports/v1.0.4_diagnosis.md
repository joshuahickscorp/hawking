# v1.0.4 Mixtral Correctness Diagnosis

Date: 2026-05-12

## Diagnostic Runs

Step B command with `"Once upon a time"` and 4 generated tokens was started
with `DISMANTLE_MIXTRAL_DEBUG=1`, but the 120s alarm ended it during prefill
because the current Mixtral path is still very slow. A shorter `"Once"` /
1-token debug run completed and captured decode-time logits.

## Key Observations

- GPU path still enters cleanly:
  `[mixtral] metal_ctx: true, weights_mmap_buf: true, embed_buf: true, decode_arena: true`
- RoPE theta is not the first bug: diagnostics print
  `rope_theta=1000000`.
- GQA attention is active with `heads=32 kv_heads=8`.
- Gate logits vary and routes are not stuck; examples include
  `routes=[(3, 0.7027148), (1, 0.29728517)]`.
- The first prompt token for `"Once"` is followed by token ids:
  `28762`, `28711`, `28717`, `28706`.
- That means `"Once"` is encoded as several byte/character fallback tokens,
  not as normal Mistral/LLaMA SentencePiece-style word/subword tokens.
- Top logits often include plausible text pieces, but the model is being
  conditioned on the wrong prompt tokenization. Example after `"Once"`:
  `top3_decoded=[(28725, ","), (297, " in"), (272, " the")]`.

## First Divergence

Tokenizer encode is the first clear semantic divergence. Mixtral's GGUF vocab
contains SentencePiece-style tokens with the `▁` word-boundary marker, but the
current `Tokenizer::from_gguf` fallback treats `tokenizer.ggml.model = "llama"`
as GPT-2 byte-level BPE. That causes normal English text to encode through byte
fallback tokens around the `287xx` range.

## Hypothesis

The Mixtral output is garbage because the prompt and generated-token stream use
the wrong tokenizer behavior. The model weights can produce plausible logits,
but the context is byte-tokenized instead of SentencePiece-tokenized.

## Next Fix Target

Implement or load a correct LLaMA/Mistral SentencePiece-compatible tokenizer
path for Mixtral. Prefer a sidecar tokenizer model/file if present; otherwise
teach the GGUF fallback enough of llama.cpp's tokenizer metadata to avoid the
byte-level BPE path for `tokenizer.ggml.model = "llama"`.
