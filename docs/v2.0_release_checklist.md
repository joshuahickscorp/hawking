## Tag procedure

When ready to tag:

```sh
# Verify clean state
git status                       # should be clean
git log --oneline -5             # recent commits all v2.0-related

# Authoritative bench (Cmd+Q Claude first)
TRIALS=10 TOKENS=64 bash tools/bench/clean_bench.sh
# Update README perf table with this number if different from current

# Tag
git -c user.name='Joshua Hicks' -c user.email='joshuahicksboba@gmail.com' \
    tag -a v2.0.0 -m "v2.0.0 — pure-Rust + Metal MoE inference engine

See CHANGELOG.md for full release notes.

Honest perf baseline:
- DeepSeek-V2-Lite Q4_K_M, M3 Pro 18 GB: <X> dec_tps (95% CI <[lo, hi]>)
- Mixtral 8×7B Q3_K_M, M3 Pro 18 GB: ~0.1 dec_tps (functional but
  SSD-bandwidth-bound on 18 GB; better on 32+ GB)

Path forward documented in CHANGELOG. Apple Neural Engine integration
is the next major architectural lever."

# Push tag
git push origin v2.0.0
```

## GitHub release

After pushing the tag, create a GitHub release:

1. Go to https://github.com/joshuahickscorp/dismantle/releases
2. "Draft a new release" → choose tag v2.0.0
3. Title: "v2.0.0 — pure-Rust MoE inference engine for Apple Silicon"
4. Description: paste the CHANGELOG.md v2.0.0 section
5. **Don't attach binaries.** Source-build via `cargo build --release` is
   the supported install path (avoids the codesigning + notarization
   complexity of distributing a pre-built Mac binary).
6. Mark as "latest release"

## After release
