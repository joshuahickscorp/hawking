# Hawking roadmap

This is the compact sequencing map. Scientific definitions live in the bound
Doctor and Appendix documents; implementation truth lives in source and
machine-readable receipts.

## Operating doctrine

Hawking optimizes three currencies together:

1. capability retained at lower physical bpw;
2. end-to-end speed and resource efficiency;
3. reproducibility strong enough to survive handoff and rerun.

No lane may buy a headline by hiding bytes, quality loss, skipped work,
contaminated timing, or unrecoverable state. Aggression means exploring more
facets and using idle resources better, while every promotion remains exact,
measured, reversible, and source-bound.

## Current sequence

### 1. Complete the active Doctor ladder

- Preserve the detached supervisor, queue, runtime specs, checkpoints, result
  roots, and source-bound binaries.
- Emit rung telemetry and intermediate analysis without treating it as final
  campaign evidence.
- Use low-bpw failures to refine routing and treatment priors, not to rewrite
  completed cells.
- Keep disk, swap, RAM, thermals, and heavy-owner gates authoritative.

### 2. Close the signed campaign boundary

Final interpretation requires terminal cells, accepted reporter checkpoints,
matching hashes and covered-cell sets, and a quiescent queue. Freeze the exact
inputs before producing a final interpretation packet. Asynchronous reporter
views do not become authority without queue acceptance.

### 3. Qualify the acceleration stack

At owner-free checkpoints:

- run complete exact 8/12/16/20 thread matrices by tier and rate;
- qualify block-parallel and ordered-pipeline paths on real artifacts;
- measure phase overlap, bounded preprocessing reuse, native CPU/PGO and I/O;
- run CPU and Metal facets separately before any stacked shared-power test;
- execute randomized matched full-stack A/B pairs with physical counters;
- promote only through a pending-only runtime generation with rollback.

### 4. Treat 120B as a mountain

The 120B lane is not a scaled-up small-model cell. It has independent source
inventory, tokenizer/architecture, shard lifecycle, storage, process-residency,
restart, and evidence constraints. Structural readiness and current disk
admissibility are separate. Component readiness cannot create a completion ETA
until every required runtime operation is reviewed.

### 5. Admit post-120B models separately

Every higher tier needs:

- immutable source and architecture authority;
- exact logical parameter count and tensor inventory;
- streamed disk and lifecycle plan;
- adapter and tokenizer support;
- representative resource receipts;
- its own quality and rollback gates.

Display labels or available downloads do not constitute admission.

### 6. Incorporate Appendix wins

The Appendix explores TQ compute-for-memory, speculative decode re-entry,
physical counters, and cross-lane compositions. Incorporate only after its
oracle and physical gates pass, and never mutate the scientific meaning of a
completed Doctor result.

### 7. Light up HIDE with the production model

The HIDE shell, event contract, local agent plumbing, and distribution surface
exist. The remaining product-value boundary is the production Hawking model
path plus deeper workspace binding, keyboard/focus polish, sandboxed execution,
and updater/onboarding closure. Keep headless state authoritative.

## HIDE product contract

HIDE is the radiating face of Hawking: nothing leaves the machine, and nothing
the agent does is hidden from the user. The product is an observatory, not a
cockpit or a chat box bolted onto a black box.

### Current state

The Planner-Executor-Verifier loop, `BackendHost`, runtime supervision, typed
intent/event transport, session replay/fork, worktree fleet primitives, code
index, context stack, chat, editor, diff review, terminal, tools, artifacts,
and workstation projections are built. The desktop artifact has been built,
signed, notarized, stapled, and smoke-tested.

The material system, typography, responsive geometry, bounded notices,
persisted view state, lazy-loaded editor, production-live transport default,
and loopback origin restriction are implemented. The product-value blocker is
the intended production `.tq` model path. Remaining work is deep workspace
re-rooting, real filesystem/git binding, keyboard/focus completion,
server-side sandboxing, first-run model selection, and update delivery.

### Wedge and minimum lovable product

The wedge is: **fork one warm project state into several approaches, run them
locally, observe every move, and keep the best**. It combines state cloning,
zero-marginal-cost local fleets, transparent progress, and hunk-level review.

The minimum lovable product includes:

- native local model serving;
- instant resume and state save/load;
- one-key fan-out and a calm lane board;
- identical hunk review in IDE and merge paths;
- a persistent Context Stack with pin/evict controls;
- a doctrine-correct shell with no fake progress;
- verifiable loopback/no-egress behavior.

Later chapters include latent state handoff, personalization, overnight
digests, multiple format tiers, and speculative execution. They do not block
the wedge.

### Product risks

- Production `.tq` serving is the load-bearing dependency.
- Coding quality must pass a real generation gate; PPL is not enough.
- Warm state saves recompute but does not guarantee recall; exact recall routes
  through indexed retrieval.
- State handoff is not shippable without an audit tap.
- Local fleets require RAM, thermal, and power admission.
- Grammar-constrained tools keep a repair fallback for hard schemas.
- Personalization remains adapter-based, MLX-trained, and held-out gated.
- A cloud escape hatch must never become the hidden reason a local demo works.

### Design doctrine

- Near-black concrete surfaces, warm bone text, and one restrained gold
  radiation edge.
- Display type is editorial; operational chrome is monospaced.
- The gold edge communicates active state, streaming, approval, fork lineage,
  and handoff. It is semantic, not decoration.
- The three primary surfaces are Chat, IDE, and Workstation; the Context Stack
  is the persistent right rail tying them together.
- Motion shows state transition and causality. No spinner substitutes for real
  progress.
- Copy is terse, calm, specific, and honest about unavailable capability.
- The interface must not be mistaken for VS Code, a generic dashboard, or a
  neon AI cockpit.

### Headless contract

The headless path is truth. The UI folds events into view state and emits typed
intents; it never becomes an independent source of authoritative state.

Core transport:

- HTTP intent submission;
- WebSocket event stream plus catch-up;
- connector calls for search, context, runtime, personalization, and research.

Core event kinds:

- projection patch;
- coalesced token batch;
- runtime status;
- tool progress;
- security gate;
- error;
- bounded custom event.

Core intents cover turn submission, cancellation, pause/resume, file open,
command execution, diff accept/reject, scrub, fork, and custom steering.
Search and other non-turn data use connectors rather than invented UI state.

### Surface map

- **Chat:** conversation, plan, tool progress, pause/cancel/resume.
- **IDE:** file tree, editor, diff review, terminal, source-control projection.
- **Workstation:** several session timelines keyed by independent session IDs.
- **Context Stack:** exact manifest of retrieved, pinned, generated, and
  budgeted context; rewinds with session replay.
- **Timeline:** causal event sequence, checkpoints, scrub, and fork.

Every panel renders a projection. Tokens are the only independent streaming
text path. Accept/reject and command effects remain backend operations.

### Security, privacy, and distribution

- Bind local services to loopback.
- Restrict CORS/origins and methods explicitly.
- Classified dangerous commands require a backend gate; front-end state cannot
  bypass it.
- Workspace writes remain confined and terminal execution routes through the
  sandboxed tool path.
- Production builds cannot silently select mock transport.
- Global errors surface as bounded notices rather than disappearing.
- Release artifacts require signing, notarization, stapling, and smoke tests.
- Auto-update requires a signed feed and is not complete until delivery and
  rollback are tested.

### Licensing

Bundled dependencies stay within the reviewed license allowlist. Monaco,
xterm, axum, portable-pty, and store dependencies retain notices. External UX
projects may be studied according to license, but reference-only sources must
not leak copied lines. Proprietary products inform observable behavior only.

## Parallel work that is safe during a detached run

- read-only analysis of receipts, queue projections, and code structure;
- isolated builds and CPU-cheap tests with bounded jobs;
- documentation and source condensation in a separate worktree;
- default-off scaffolds that do not import active runtime state;
- static model/source inventories and contract generation;
- test parameterization and shared utility extraction with parity gates.

Avoid model downloads, heavy corpus reads, Metal dispatch, broad rebuilds, or
anything that changes the active checkout unless the owner and resource gates
explicitly permit it.

## Repository condensation

The condensed repository has one canonical document per subject, one CLI
surface per domain, shared evidence primitives, policy-driven supervisors, and
parameterized tests. Git history is the archive. Growth budgets should reject:

- dated handoffs merged as permanent documentation;
- generated reports or local build products;
- new standalone entrypoints that duplicate an existing command;
- repeated canonical JSON, hashing, locking, or atomic-write implementations;
- compatibility wrappers with no scheduled removal gate.

## Completion gates

- exact output or declared approximate-quality parity;
- equal or better collected test coverage;
- no material throughput, RSS, swap, or disk regression;
- crash/resume and rollback proof;
- no broken active documentation links;
- explicit supersession records for source-bound paths;
- a clean worktree containing no generated evidence.
