## Accepted defects

Accepted for this preview, with reasons. None of these is a reason the preview is blocked; the blockers are the five CRITICALs and the nine MAJORs above, and none of those appears here.

1. **`fleet` projection has no producer; `FleetView` and the background-work chip are dead on live.** Accepted: `FleetView.tsx:4-5` states this in its own header. An unbuilt producer that labels itself as unbuilt is the correct handling; nothing false is displayed.
2. **`build` and `test` projections bound by `ContextStack.tsx:104-105` with no writer.** Accepted: the render is guarded by `{build || test ? ...}`, so the block is unreachable rather than false. Dead render code, not a lie.
3. **`onboarding.ts` keeps three exports with no production caller.** Accepted: the closeout forbids feature work, the gallery records the defect openly, and the exports are inert. Delete them in the next build, not this one.
4. **Loopback transport has no authentication and gate ids are an enumerable counter.** Accepted as normal for a local dev transport. It does mean the gate constrains the in-process agent only, not the machine, and it must not be accepted once the transport is reachable from anywhere but 127.0.0.1.
5. **ACP and SDK route parity is vacuous.** Accepted: both are `DEFERRED_MODEL_REQUIRED` and neither is bound to `BackendHost`, so there is no effect to police. This becomes a blocker the moment either surface gains a real turn handler; the parity claim must be re-earned then, not carried forward.
6. **Thread coverage in checkpoints is unproven (count 0).** Accepted: no model is served on the preview host, so no transcript exists to cover. Structurally unprovable, not a defect in the code under review.
7. **Checkpoint `repo_state` digests keyed on absolute paths.** Accepted for a single-root preview. It makes every prior checkpoint report coverage drift if the same workspace is mounted at a different root; fix alongside the relative-vs-absolute CRITICAL, since both are the same root cause.
8. **`compare_to_checkpoint` skips `verify_integrity`.** Accepted because compare is read-only and its two write-capable siblings both refuse a tampered record. Still inconsistent; it should call the same helper.
9. **Eight of ten timeline verbs ignore the selected dot.** Accepted as a scoped functional gap, not a safety defect: the row's label promises more than `timelinePlan` delivers, but no verb acts on an address the user did not create. The label is the thing to change if the behaviour is not.
10. **`VITE_HIDE_TRANSPORT=mock` can still be forced into a production build.** Accepted as a build-time footgun with no runtime exposure: a mock bundle talks to no backend and touches no workspace. Add one CI assertion when convenient.
11. **Ctrl+P / Ctrl+B collide with Cocoa caret bindings on macOS.** Accepted because nothing bound to a Mod chord at the shell level carries a `write_fs` effect; the worst outcome is a palette opening while typing. Not accepted as a pattern: the same window-level listener would be a safety defect the day a destructive command gains a shortcut.
12. **`Cargo.toml` carries one unrelated comment rewrite in `vendor/strand-quant`.** Accepted as cosmetic, in a comment, in a crate outside the default workspace build. It is still scope creep in a file that was supposed to change for one reason.
13. **Nothing is committed; 234 paths are working tree only.** Accepted for an internal adversarial pass. NOT acceptable for an owner preview: an owner cannot review, bisect or revert a tree git has no record of. Commit before any preview, or the preview has no baseline.
