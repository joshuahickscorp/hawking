## L4 attempt details (informational — not committed)

Tried: pin kv_b_proj as f16 + new `mla_decode_kernel_f16w` shader variant, opt-in via
new `mla_kv_b_dtype` profile field. Build clean, parity bit-identical (12 tokens).

Why it failed to gate:
- mla_decode_kernel reads 32 KB of kv_b_proj per call per head (the head-local slice).
  Apple GPU L1 / L2 cache holds this easily; halving DRAM weight bytes saved < 1% wall.
- Compare to T2.13: q_proj reads 25 MB per call, which is too large to fit in cache and
  hit DRAM bandwidth ceiling. Halving paid 6.6%.
- **Lesson:** weight-bandwidth optimization only pays when the read set exceeds cache
  capacity. Apply this filter before attempting future `_f16w` ports.

### Session A: L7 (measurement infra)

Wire `MTLCounterSampleBuffer` into the production single-CB path so per-kernel GPU
times are measurable on the REAL decode CB (not the split-CB diagnostic mode, which
inflates absolutes by ~20%). This is the foundation for evidence-driven kernel
selection going forward. No bench gate (it's tooling).

Reference: `crates/dismantle-core/src/metal/mod.rs:114-124` is the existing GPU-time
read via `GPUStartTime/EndTime` on a whole CB — extend to per-dispatch via
`MTLCounterSampleBuffer`. Apple docs:
https://developer.apple.com/documentation/metal/mtlcountersamplebuffer

Touch points:
- Add `MTLCounterSampleBuffer` allocation to `MetalContext` (alongside `DispatchTrace`)
- Add `sampleCountersInBuffer:atSampleIndex:withBarrier:` calls before+after each
  compute dispatch via `dispatch_threads`
- After CB completes, read sample buffer; convert ticks → us; populate
  `DispatchSample::gpu_us`
- Add `DISMANTLE_TCB_TRACE=gpu_prod` mode that activates the counter sampling without
  splitting the CB
- Update `tools/bench/analyze_tcb_trace.py` to label which mode produced a trace and
  flag absolute% accuracy

Estimated size: 200-400 lines of careful Metal + ObjC msg_send (the `metal` 0.29
crate doesn't wrap `MTLCounterSampleBuffer` either, same as `GPUStartTime`).
Estimated time: half-to-full session.
